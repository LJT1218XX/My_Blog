构建一个**个人博客主界面**，整体采用 **Apple 设计语言**——极简美学，通过大量留白、精致圆角、柔和阴影、毛玻璃质感及克制的色彩，营造高级、宁静、专注的阅读氛围。

### 核心目标
- 呈现博主个人信息与品牌形象
- 展示文章列表，支持浏览与快速筛选
- 提供清晰的导航与内容发现路径
- 传递简洁、优雅、专业的视觉感受

### 交付范围
- 单一 HTML 页面（含 CSS、JavaScript）
- 完整的导航栏、Hero 区、文章卡片列表、侧边栏、页脚
- 响应式适配：桌面端（≥1024px）、平板（768-1024px）、手机（<768px）

## 3. 页面布局结构

桌面端采用两栏布局（主内容 + 侧边栏），移动端单列堆叠。

### 桌面端布局 (≥1024px)
```
┌─────────────────────────────────────────────┐
│  固定导航栏 (毛玻璃, sticky)                  │
├─────────────────────────────────────────────┤
│           Hero 区 (头像、简介、社交)          │
├──────────────────────────┬──────────────────┤
│   主内容区 (文章卡片列表) │   侧边栏          │
│                          │ (关于我/热门/标签) │
├──────────────────────────┴──────────────────┤
│              页脚 (版权、友链)               │
└─────────────────────────────────────────────┘
```
- 主内容区与侧边栏比例约 3:1
- 侧边栏 sticky 定位
- 最大内容宽度 1200px 左右，居中

### 移动端布局 (<768px)
- 导航栏改为汉堡菜单
- 所有区块垂直堆叠
- 侧边栏内容移至页脚上方或折叠展示

## 4. 模块详细需求

### 4.1 固定导航栏 【优先级：高】
- 高度：桌面 52px，移动 48px
- 左侧：站点名称/Logo
- 中间：导航链接（首页、文章、关于、标签）
- 右侧：搜索图标或主题切换按钮
- 移动端：汉堡菜单展开全屏导航，带弹性动画
- 滚动行为：向下滚动时背景不透明度从 0.72 渐变至 0.9，增加底部细线分隔

### 4.2 Hero 个人信息区 【优先级：高】
- 头像：圆形，直径 88–96px，带阴影 `0 12px 32px rgba(0,0,0,0.08)`，悬停轻微放大
- 昵称：48–56px 粗体
- 简介：18–20px，次级文字色
- 社交图标：GitHub、Twitter、Email 等 SVG，悬停变蓝
- 动画：页面加载时从下方 40px 淡入上移 (fade-in-up)

### 4.3 文章卡片列表 【优先级：高】
- 每张卡片包含：可选封面图、标题、摘要（2行截断）、日期、标签、阅读时长
- 卡片间距 24px
- 卡片样式：白色背景，圆角 18px，阴影 `0 4px 12px rgba(0,0,0,0.05)`
- 悬停效果：卡片上浮 `translateY(-4px)`，阴影变为 `0 12px 32px rgba(0,0,0,0.08)`，标题变蓝
- 标签：药丸形小标签，浅背景色（如 `#e8f4fd` 配 `#0071e3`）
- 底部提供“加载更多”按钮或实现无限滚动

### 4.4 侧边栏 【优先级：中】
- 宽度：300–320px
- 模块：“关于我”卡片、“热门文章”列表、“标签云”
- 粘性定位，跟随滚动
- 移动端：内容移至页脚区域

### 4.5 页脚 【优先级：低】
- 简约设计，版权信息、友情链接、回到顶部按钮
- 背景色略深于页面背景 (`#e8e8ed` 或类似)

## 6. 响应式设计策略

定义三个断点：

| 断点        | 布局         | 导航       | 侧边栏       | 字体缩放     |
|-------------|--------------|------------|--------------|--------------|
| <768px      | 单列堆叠     | 汉堡菜单   | 移至底部     | 标题缩至 70% |
| 768-1024px  | 两列 (3:2)   | 完整链接   | 右侧显示     | 标题缩至 85% |
| ≥1024px     | 两列 (3:1)   | 完整链接   | 右侧 sticky  | 100% 标准    |

- 优先移动端，使用 `@media` 渐进增强
- 图片使用 `max-width: 100%` 自适应

## 7. 技术栈建议

- **基础：** HTML5 + CSS3 + 原生 JavaScript（无框架依赖）
- **CSS 方案：** CSS 自定义属性 (变量) + Flexbox + Grid；可选 Tailwind CSS
- **图标：** SF Symbols 风格 SVG 或 Lucide Icons
- **字体：** 系统字体栈优先
- **可选交互增强：** Alpine.js 或 Petite-Vue

## 8. 实现注意事项

- 导航栏毛玻璃效果需使用 `backdrop-filter`，注意 Safari 支持性。
- 文章卡片数据可先用静态数组模拟，方便后续对接 API。
- 响应式图片使用 `srcset` 或 CDN 动态裁剪。
- 所有动画使用 CSS transition/animation，减少 JS 操作 DOM。
- 移动端汉堡菜单需处理 `body` 滚动锁定。
- 代码组织：推荐将 CSS 变量集中定义，组件样式模块化。
